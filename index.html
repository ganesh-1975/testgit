Git Commands and their uses

Git is a version control system which needs to be installed in our machines to keep track of the code file changes and later save them on a remote repository like Github or Bigbucket.

W3school Page for Git and GitHub

Git command and and their functions

  • git init

    It is used to intialize git to do version control for our project in our working directory.

  • git --version

    It is used to check the git version installed in our system.

  • git config --global user.email "you@example.com"

    It is used to set the user email id which is used as identification while commiting the code.

  • git config --global user.name "Your Name"

    It is used to set the user name which is used as identification while commiting the code.

  • git status

    It is used to know the sattus of our code files like whether they are untracked,staged or commited.

  • git add [filename]

    It is used to stage the untracked code file. Means the file from the working directory is moved to the staging area where this file is held for its changes to be commited to local repository.

  • git add .

    It will add i.e. stage all the code files to the staging area.

  • git commit -m "[message]"

    It is used to commit the staged changes to the local repository with a -m option where the message is given to identify the recent changes. Basically we are making a log of the changes.

  • git log

    It is used to check the list of commits we have done so far.

  • git reset --hard [SHA number]

    [SHA number is the secured hash algorithm number which is unique for each commit.] It is used to delete the recent commit change and go back to the previous commit working code represented by that particular commit's SHA.

  • git reset --soft [SHA number]

    It is used to go back to the previous commited code along with the latest code change still available.The git log will only show previous commited logs but the recent commited log will not be there even though the change in this commit will be still available.

  • git remote add origin [url]

    It is used to set the origin location on remote repository where the code needs to be pushed from local repository.

  • git branch -M main

    This sets the branch name as main

  • git push -u origin main

    It is used to push the code from local to remote at the specified URLs origin's main branch .

  • git clone url

    It is used to get the code from remote to local .

  • git pull

    Here u need to go into the folder where the code was cloned and do git pull . It is used to get any latest changes which happened on remote. Actually git pull has two steps in it. git fetch which fetches the code from remote and git merge which merges any new changes to the local .

  • git branch

    It is used to check which are the branches and which is the current branch .

  • git branch [branchname]

    It is used to create a branch with the mentioned branch name .

  • git checkout branchname

    It is used to check out from current branch and move on to the mentioned branch .

  • git merge branchname

    It is used to merge the new changes done in the mentioned branchname into the current branch.
